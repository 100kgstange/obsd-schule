02 ✅ CPU

TARGET DECK: [Schule] Prüfung::💽Hardware::01 ✅ CPU

Funktionsweise einer CPU: #flashcard

  • Die CPU (Central Processing Unit) ist das Herzstück eines Computers und führt die meisten Berechnungen durch.
  • Sie besteht aus drei Hauptteilen: der Steuerlogik, dem Rechenwerk und dem Registersatz.
  • Die CPU führt Befehle aus, die in Maschinensprache geschrieben sind.
  • Der Befehlszyklus besteht aus Fetch (Abrufen), Decode (Dekodieren), Execute (Ausführen) und Store (Speichern).
Befehlszyklus CPU: #flashcard
  • Die Ausführung eines Befehls durch die CPU durchläuft verschiedene Phasen. Ein solcher Durchlauf wird als Befehlszyklus bezeichnet.
  • Liegt eine von Neumann Architektur vor, wird dieser Befehlszyklus in **fünf Phasen eingeteilt (Von Neumann Zyklus).
  • Liegt eine Harvard Architektur vor können zwei der fünf Phasen zusammengefasst werden. Der Befehlszyklus besteht dann aus vier Phasen.
Neumann Zyklus: #flashcard
  1. FETCH
    Die Instruktion (Maschinenbefehl) muss aus dem Speicher geladen werden. Dies bedeutet, dass auf dem Datenbus der Maschinencode für den Befehl liegt.
  2. DECODE
    Der Maschinenbefehl muss nun von der CU (= Central Unit) decodiert werden. Diese gibt über den Steuerbus die Funktion der ALU und ggf. weiterer Komponenten vor.
  3. FETCH OPERANDS
    In dieser Phase werden ggf. für die Ausführung des Befehls benötigte Operanden in die dafür vorgesehenen Register geladen.
  4. EXECUTE
    Der Befehl wird von der ALU ausgeführt. (ALU = Arithmetic Logic Unit)
  5. WRITE BACK
    Die Ergebnisse der Berechnung werden in den Speicher geschrieben.
Harvard Zyklus: #flashcard
  • Für die Harvard Architektur können die Phasen 1 und 3 des von Neumann Zyklus zusammengefasst werden.
  • Operanden und Instruktionen können gleichzeitig geladen werden, da unterschiedliche Bus-Systeme involviert sind.
  • Der Befehlszyklus hat also nur vier Phasen.
  1. FETCH
    Die Instruktion (Maschinenbefehl) muss aus dem Programmspeicher geladen werden. Aus dem Datenspeicher werden die Operanden in die dafür vorgesehenen Register geladen.
  2. DECODE
    Der Maschinenbefehl muss nun von der CU decodiert werden. Diese gibt über den Steuerbus die Funktion der ALU und ggf. weiterer Komponenten vor.
  3. EXECUTE
    Der Befehl wird von der ALU ausgeführt.
  4. WRITE BACK
    Die Ergebnisse der Berechnung werden in den Speicher geschrieben.

CPU-Registersatz: #flashcard

  • Register sind kleine Speicherbereiche direkt in der CPU, die für die Zwischenspeicherung von Daten und Adressen verwendet werden.
  • Beispiele sind das Programmzähler-Register (PC), das Befehlszähler-Register (IR) und das Akkumulator-Register.

CPU: Arten von Befehlen: #flashcard

Befehle können arithmetische, logische, Sprung- oder Kontrollbefehle sein.

  • Arithmetische Befehle: Diese führen mathematische Operationen durch, um numerische Werte zu manipulieren. Sie werden in Anwendungen wie Berechnungen, Finanzwesen, Physiksimulationen und anderen mathematischen Verarbeitungen verwendet.
  • Logische Befehle: Logische Befehle führen Operationen auf der Ebene von Bits durch. Sie werden für das Setzen und Überprüfen von Bedingungen verwendet. Logische Befehle sind wichtig für die Steuerung des Programmflusses und das Erstellen von bedingten Anweisungen.
  • Sprungbefehle können dazu verwendet werden, um zu einem anderen Abschnitt des Programms zu springen, wenn eine Variable einen bestimmten Wert erreicht.
  • Kontrollbefehle steuern den Ablauf des Programms, indem sie die Ausführung von Befehlen beeinflussen. Hierzu gehören beispielsweise Befehle für den Programmstart, das Beenden oder die Unterbrechung der Ausführung.

CPU: Pipelining: #flashcard

  • Pipelining ist eine Technik, die in modernen CPUs verwendet wird, um die Effizienz der Befehlsausführung zu verbessern.
  • Der Ausführungsprozess wird in mehrere Stufen unterteilt, und verschiedene Befehle werden gleichzeitig in verschiedenen Stufen bearbeitet.
  • Jede Stufe führt eine spezifische Aufgabe aus, und während eine Instruktion durch die Pipeline geht, kann die nächste Instruktion in die nächste Stufe eintreten. Dies ermöglicht eine Überlappung der Befehlsausführung und somit eine potenzielle Steigerung der Gesamtausführungsgeschwindigkeit.

Die typischen CPU Pipelinestufen sind: #flashcard

  1. Befehlsaufnahme (Instruction Fetch)
  2. Dekodierung (Instruction Decode)
  3. Ausführung (Execution)
  4. Speicherzugriff (Memory Access)
  5. Schreibrücklauf (Write Back)

CISC vs. RISC: #flashcard

CISC (Complex Instruction Set Computing) und RISC (Reduced Instruction Set Computing) sind zwei unterschiedliche Architekturansätze für CPUs. Während CISC einen komplexen Befehlssatz bietet, setzt RISC auf eine vereinfachte, effiziente Struktur.

CISC: #flashcard

  • CISC-Architekturen haben einen umfangreichen Satz komplexer Befehle.
  • Ein einzelner Befehl in CISC kann mehrere Operationen durchführen, wodurch der Befehlssatz größer und leistungsfähiger ist.
  • Ermöglicht direkt adressierbare Speicherzugriffe innerhalb eines einzelnen Befehls.
  • Wird häufig in x86- und x86-64-Prozessoren von Intel und AMD verwendet.
  • Neigt dazu, mehr Energie und Transistoren für die Dekodierung der komplexen Befehle zu verbrauchen.
  • Beispiel-Architekturen: Intel x86, AMD x86-64, Motorola 68000

RISC: #flashcard

  • RISC-Architekturen haben einen einfachen Satz von grundlegenden Befehlen, die schneller und effizienter verarbeitet werden können.
  • Jeder Befehl in RISC führt eine einfache Operation aus, wodurch der Befehlssatz kleiner und optimiert ist.
  • Meistens wird eine Load/Store-Architektur verwendet, d. h., Speicherzugriffe erfolgen nur über separate Lade- und Speicherbefehle.
  • Wird häufig in ARM, RISC-V, PowerPC und SPARC-Prozessoren verwendet, die für mobile Geräte und Hochleistungsserver optimiert sind.
  • Geringerer Energieverbrauch und höhere Effizienz im Vergleich zu CISC, besonders vorteilhaft für Embedded-Systeme und Mobilgeräte.
  • Beispiel-Architekturen: ARM (z. B. Apple M1/M2), RISC-V, IBM Power, MIPS

CPU: Cache-Speicher: #flashcard

  • Der Cache-Speicher ist ein kleiner, aber schneller Speicherbereich, der zwischen dem Hauptprozessor und dem langsameren Hauptspeicher liegt.
  • Seine Hauptaufgabe ist es, häufig verwendete Daten und Instruktionen zu speichern, um den Zugriff auf diese Informationen zu beschleunigen.

Einige Schlüsselaspekte des Cache-Speichers: #flashcard

  • L1-Cache: Der L1-Cache ist der schnellste, direkt in den Prozessor integrierte Cache. Er ist in der Regel in separate Instruktions- und Daten-Caches unterteilt.
  • L2-Cache: Der L2-Cache ist größer als der L1-Cache und befindet sich normalerweise auf demselben Chip wie der Prozessor. Er dient als Puffer zwischen dem L1-Cache und dem Hauptspeicher.
  • L3-Cache: Der L3-Cache ist noch größer und befindet sich auf dem Prozessor oder in unmittelbarer Nähe. Er wird von mehreren Prozessorkernen gemeinsam genutzt.
  • Cache-Austauschalgorithmen: Hierbei handelt es sich um Mechanismen, die festlegen, welche Daten im Cache gespeichert werden sollen und welche entfernt werden sollen. Gängige Algorithmen sind beispielsweise "Least Recently Used" (LRU) und "Random Replacement".

  1. Clock Speed und Performance: #flashcard

    • Die Taktgeschwindigkeit (Clock Speed) einer CPU beeinflusst die Verarbeitungsgeschwindigkeit.
    • Jedoch ist eine höhere Taktgeschwindigkeit nicht immer gleichbedeutend mit besserer Leistung aufgrund von Architekturunterschieden.

  2. Multithreading und Multicore: #flashcard

    • Moderne CPUs unterstützen Multithreading (mehrere Threads gleichzeitig) und Multicore (mehrere physische Kerne auf einem Chip) für verbesserte Leistung.

  3. Prozessorarchitekturen: #flashcard

  • x86, ARM und andere Architekturen sind verbreitet. Jede hat ihre eigenen Merkmale und wird in verschiedenen Geräten eingesetzt.

Prozessorarchitekturen

Es gibt verschiedene Prozessorarchitekturen, die in verschiedenen Arten von Computern und elektronischen Geräten verwendet werden. Hier sind einige der wichtigsten:

  1. x86-Prozessorarchitektur: #flashcard

    • Diese Architektur wird am häufigsten in Desktop- und Laptop-Computern verwendet.
    • Intel und AMD sind die wichtigsten Hersteller von x86-Prozessoren.
    • Bekannte Befehlssätze sind x86-32 (32-Bit) und x86-64 (64-Bit).

  2. ARM-Prozessorarchitektur: #flashcard

    • ARM (Acorn RISC Machine) ist eine RISC-Architektur, die in vielen mobilen Geräten wie Smartphones, Tablets und eingebetteten Systemen verwendet wird.
    • ARM-Prozessoren sind bekannt für ihre Energieeffizienz.
    • Sie werden auch in einigen Servern und zunehmend in Laptops verwendet.

  3. MIPS-Prozessorarchitektur: #flashcard

    • MIPS (Microprocessor without Interlocked Pipeline Stages) ist eine RISC-Architektur, die in der Vergangenheit in Workstations, Servern und eingebetteten Systemen verwendet wurde.
    • Sie hat jedoch an Bedeutung verloren.

  4. Power-Prozessorarchitektur: #flashcard

    • Ursprünglich von IBM entwickelt, wird die Power-Architektur in Hochleistungsrechnern, Servern und einigen IBM-Systemen eingesetzt.
    • Apple hat auch angekündigt, auf die eigene ARM-Architektur umzusteigen, während sie vorher auf PowerPC basierten.

  5. SPARC-Prozessorarchitektur: #flashcard

    • SPARC (Scalable Processor Architecture) ist eine RISC-Architektur, die hauptsächlich in Servern und Workstations von Oracle verwendet wird.

  6. RISC-V-Prozessorarchitektur: #flashcard

    • RISC-V ist eine offene, lizenzfreie RISC-Architektur, die an Bedeutung gewinnt.
    • Sie wird in verschiedenen Anwendungen von eingebetteten Systemen bis hin zu High-Performance-Computing (HPC) eingesetzt.

  7. Alpha-Prozessorarchitektur: #flashcard

    • Die Alpha-Architektur wurde von DEC (Digital Equipment Corporation) entwickelt und wurde früher in Workstations und Servern eingesetzt. DEC wurde später von Compaq übernommen, und die Alpha-Architektur wurde eingestellt.

  8. Itanium (IA-64) Prozessorarchitektur: #flashcard

    • Itanium ist eine von Intel und HP entwickelte Architektur für Hochleistungs-Server.
    • Es hat jedoch keine breite Akzeptanz gefunden, und die Produktion wurde eingestellt.

Diese Architekturen variieren in ihrer Anwendung, Leistung und Energieeffizienz. Die Wahl der Architektur hängt oft von den spezifischen Anforderungen der Geräte oder Systeme ab, in denen sie eingesetzt werden.

CPU: 32-bit vs 64-bit #flashcard

  • Die Bezeichnungen "64-Bit" und "32-Bit" beziehen sich auf die Breite eines Prozessorregisters und darauf, wie viele Datenbits er in einem einzigen Schritt verarbeiten kann.
  • In der heutigen Zeit werden viele Computer und Betriebssysteme als 64-Bit-Systeme entwickelt, um von den Vorteilen der größeren Adressierbarkeit und Datenverarbeitung zu profitieren.
  • Bei der Wahl zwischen 32-Bit- und 64-Bit-Systemen ist es wichtig zu beachten, dass die Softwarekompatibilität eine entscheidende Rolle spielt.

Hier sind die Hauptunterschiede zwischen 64-Bit- und 32-Bit-Systemen: #flashcard
9. Registerbreite:

  • 32-Bit: Ein 32-Bit-Prozessor kann in einem einzigen Taktzyklus Daten mit einer Breite von 32 Bits verarbeiten.
  • 64-Bit: Ein 64-Bit-Prozessor kann Daten mit einer Breite von 64 Bits in einem einzigen Taktzyklus verarbeiten.
  1. Adressraum:
  • 32-Bit: Ein 32-Bit-System kann theoretisch auf 2^32 (ungefähr 4 GB) Speicheradressen zugreifen.
  • 64-Bit: Ein 64-Bit-System kann auf 2^64 Speicheradressen zugreifen, was theoretisch eine extrem große Menge an Speicher ermöglicht (mehr als derzeit praktisch verfügbar). (= 16 Exabyte (EB) oder = 16 Millionen Terabyte (TB) oder = 16 Milliarden Gigabyte (GB))
  1. Leistung und Datenverarbeitung:
  • 64-Bit: Durch die Verarbeitung von Daten in 64-Bit-Chunks kann ein 64-Bit-System theoretisch größere Mengen an Daten in einem Schritt verarbeiten, was in bestimmten Anwendungen wie Videobearbeitung oder wissenschaftlichen Berechnungen von Vorteil sein kann.
  • 32-Bit: Obwohl 32-Bit-Systeme in vielen Anwendungen immer noch effizient arbeiten können, können sie aufgrund der begrenzten Adressierbarkeit in bestimmten Szenarien, insbesondere bei großen Datenmengen, Einschränkungen haben.
  1. Softwarekompatibilität:
  • 32-Bit: 32-Bit-Software läuft nativ auf 32-Bit-Systemen, während 64-Bit-Software normalerweise nicht kompatibel ist. Viele moderne Betriebssysteme und Anwendungen bieten jedoch sowohl 32-Bit- als auch 64-Bit-Versionen an.
  • 64-Bit: 64-Bit-Systeme können sowohl 32-Bit- als auch 64-Bit-Software ausführen.
  1. Sicherheit und Stabilität:
  • 64-Bit: In 64-Bit-Systemen können mehrere Sicherheitsfunktionen, wie beispielsweise das DEP (Data Execution Prevention), besser implementiert werden, was die Systemsicherheit verbessern kann.
  • 32-Bit: Ältere 32-Bit-Systeme könnten in bestimmten Sicherheitsaspekten weniger fortschrittlich sein.

Definition Übertaktung #flashcard

Übertaktung bezeichnet das Erhöhen der Taktfrequenz eines Prozessors über die vom Hersteller festgelegten Spezifikationen hinaus, um eine höhere Leistung zu erzielen.

Definition Taktfrequenz #flashcard

  • Die Taktfrequenz gibt an, wie schnell ein Prozessor arbeiten kann, gemessen in Hertz (Hz).
  • Eine höhere Taktfrequenz bedeutet, dass der Prozessor mehr Anweisungen pro Sekunde ausführen kann.

Definition Prozessorserie #flashcard

  • Dies ist eine Familie oder Produktlinie, die unter einem gemeinsamen Namen veröffentlicht wird und für ähnliche Aufgaben gedacht ist.
  • Bei Intel gibt es zum Beispiel die Serien Core i3, i5, i7, i9, und bei AMD die Ryzen 3, 5, 7, 9.
  • Je höher die Zahl der Serie (i5 ist besser als i3, Ryzen 7 ist besser als Ryzen 5), desto höher ist in der Regel die Leistung.
  • Die Serie gibt die Klasse des Prozessors an: Budget-, Mittel- oder Oberklasse.

Definition Generation von Prozessoren #flashcard

Dies ist eine Version einer bestimmten Serie, die im Vergleich zu vorherigen Generationen auf neueren Technologien basiert und in der Regel bessere Leistung und Energieeffizienz bietet.

CPU Spezielle Suffixe #flashcard

  • K (Intel) – freigeschaltet für Übertaktung.
  • H (Intel) – für leistungsstarke Laptops (Grafik, Spiele).
  • F (Intel) – erfordert eine dedizierte Grafikkarte, da keine integrierte GPU vorhanden ist.
  • G (Intel) – enthält leistungsstarke integrierte Grafikprozessoren.
  • T (Intel) – energieeffiziente Desktop-Prozessoren mit niedrigerer Taktfrequenz.
  • X (AMD) – erhöhte Leistung.
  • G (AMD) – integriert eine Radeon-Grafikeinheit, nützlich für leichtere Grafikanwendungen ohne separate Grafikkarte.
  • U – energieeffizient (häufig in Laptops).

Beispiel Intel Prozessorname #flashcard

Intel Core i5-13600K Die Serie ist i5 (mittleres Leistungsniveau), die Generation ist die 13. Das „K“ bedeutet, dass der Prozessor übertaktbar ist.

Beispiel AMD Prozessorname #flashcard

AMD Ryzen 7 5800XDie Serie ist Ryzen 7 (hohes Leistungsniveau), die Generation ist die 5. Das „X“ bedeutet, dass der Prozessor eine höhere Leistung bietet.

Thermal Management: Definition #flashcard

Thermal Management bezieht sich auf die Techniken und Strategien, die eingesetzt werden, um die Temperatur von Geräten, insbesondere von elektronischen Komponenten wie CPUs und GPUs, innerhalb sicherer Betriebsgrenzen zu halten.

Wichtigkeit des Thermal Managements #flashcard

Ein effektives Thermal Management ist entscheidend, um die Leistung zu maximieren, die Lebensdauer von Komponenten zu verlängern und das Risiko von Überhitzung und damit verbundenen Schäden zu minimieren.

Methoden des Thermal Managements #flashcard

  1. Passive Kühlung: Verwendung von Kühlkörpern und Wärmeleitmaterialien, um die Wärme von Komponenten abzuleiten.
  2. Aktive Kühlung: Einsatz von Lüftern, Pumpen oder anderen aktiven Kühlsystemen, um die Luftzirkulation und den Wärmeabtransport zu erhöhen.
  3. Wärmeleitung: Verwendung von Materialien, die Wärme effizient leiten, um die Temperaturverteilung zu optimieren.
  4. Wärmeabfuhr: Techniken wie Flüssigkeitskühlung oder thermische Schnittstellenmaterialien (TIM) zur Verbesserung des Wärmeübergangs.

Thermal Throttling #flashcard

Thermal Throttling ist eine Schutzmaßnahme, bei der die CPU oder GPU ihre Leistung reduziert, um Überhitzung zu vermeiden. Dies kann die Systemleistung beeinträchtigen.

Thermal Paste #flashcard

Thermal Paste wird zwischen der CPU/GPU und dem Kühlkörper aufgetragen, um die Wärmeübertragung zu verbessern. Sie füllt Mikroritzen und sorgt für eine bessere Wärmeleitung.

Overclocking: Definition #flashcard

  • Overclocking bezeichnet das Erhöhen der Taktfrequenz eines Prozessors oder einer Grafikkarte über die vom Hersteller empfohlenen Spezifikationen, um eine höhere Leistung zu erzielen.
  • Overclocking kann zu signifikanten Leistungsgewinnen führen, birgt jedoch auch Risiken. Eine sorgfältige Planung und das Verständnis der Hardware sind entscheidend für den Erfolg.

Risiken des Overclocking #flashcard

  1. Überhitzung: Erhöhte Taktfrequenzen können zu Überhitzung führen, wenn das Thermal Management unzureichend ist.
  2. Instabilität: Das System kann instabil werden, was zu Abstürzen oder Fehlfunktionen führen kann.
  3. Garantieverlust: Viele Hersteller erlöschen die Garantie, wenn das Produkt übertaktet wird.
  4. Hardware-Schäden: Langfristiges Overclocking kann zu physischen Schäden an der Hardware führen.

Belohnungen des Overclocking #flashcard

  1. Erhöhte Leistung: Overclocking kann die Leistung in Spielen und rechenintensiven Anwendungen erheblich steigern.
  2. Bessere Benchmark-Ergebnisse: Enthusiasten nutzen Overclocking, um bessere Ergebnisse in Benchmarks zu erzielen.
  3. Optimierung von Preis-Leistungs-Verhältnis: Ältere oder günstigere Hardware kann durch Overclocking konkurrenzfähig gemacht werden.

Cooling Solutions for Overclocking #flashcard

  1. Luftkühlung: Verwendung von Hochleistungs-Lüftern und Kühlkörpern.
  2. Wasserkühlung: Verwendung von Flüssigkeitskühlsystemen für bessere Kühlleistung.
  3. Phase-Change-Kühlung: Eine extrem effiziente, aber teure Methode zur Wärmeabfuhr.

Stabilitätstest nach Overclocking #flashcard

Nach dem Overclocking sollte das System durch Stabilitätstests (z.B. Prime95, AIDA64) geprüft werden, um sicherzustellen, dass es unter Last stabil bleibt und keine Überhitzung auftritt.
