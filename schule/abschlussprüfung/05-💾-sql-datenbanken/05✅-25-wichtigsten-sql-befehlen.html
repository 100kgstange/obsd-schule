1. SELECT ❗❗ #flashcard

Die SELECT-Anweisung wird verwendet, um anzugeben, welche Spalten einer Datenbanktabelle im Ergebnis enthalten sein sollen.
In diesem Beispiel werden nur die Spalten movie_name und director ausgewählt; andere Spalten werden nicht zurückgegeben.

SELECT movie_name, director
FROM films;

Um alle Spalten einer Tabelle auszuwählen, können Sie SELECT * (ausgesprochen als "Stern") verwenden. Dieses Beispiel wählt alle Spalten aus der Tabelle films aus.

SELECT * 
FROM films;

2. LIMIT #flashcard

Die Begrenzung der Anzahl der aus einer Tabelle zurückgegebenen Zeilen ist ein nützlicher Trick, um die Abfragezeit zu verkürzen. Um die Anzahl der Zeilen zu begrenzen, kann das LIMIT-Kommando verwendet werden. In diesem Beispiel werden alle Spalten aus der Tabelle films ausgewählt und dann die Ergebnisse auf die ersten zehn Zeilen begrenzt.

SELECT *
FROM films
LIMIT 10;

3. AS❗❗ #flashcard

Um eine Spalte oder Tabelle beim Zurückgeben von Ergebnissen umzubenennen, kann das AS-Kommando verwendet werden, um einen Alias für die Ausgaben festzulegen. In diesem Beispiel wird die Spalte movie_name ausgewählt und als Alias den Namen movie_title zugewiesen.

SELECT movie_name AS movie_title
FROM films;

4. SELECT DISTINCT #flashcard

Datensätze enthalten oft doppelte Zeilen oder Werte in einer Spalte. Durch die Kombination von SELECT mit DISTINCT werden Duplikate entfernt. Dieses Beispiel gibt die eindeutigen Werte in der Spalte director zurück.

SELECT DISTINCT director
FROM films;

5. COUNT ❗❗ #flashcard

COUNT() gibt die Anzahl der Zeilen in der Tabelle oder Gruppe zurück. Dieses Beispiel gibt die Anzahl der Zeilen in der Tabelle films zurück und benennt das Ergebnis dann als number_of_films um.

SELECT COUNT(*) AS number_of_films
FROM films;

6. MIN ❗❗ #flashcard

MIN() gibt den minimalen Wert in einer numerischen Spalte zurück. Für Textspalten gibt MIN() den ersten Wert alphabetisch zurück. Dieses Beispiel gibt den Film mit dem geringsten Umsatz zurück.

SELECT MIN(revenue) AS minimum_revenue,
FROM films;

7. MAX ❗❗ #flashcard

MAX() gibt den maximalen Wert in einer numerischen Spalte zurück. Für Textspalten gibt MAX() den letzten Wert alphabetisch zurück. Dieses Beispiel gibt den Film mit dem höchsten Umsatz zurück.

SELECT MAX(revenue) AS maximum_revenue,
FROM films;

8. SUM ❗❗ #flashcard

SUM() gibt die Summe der numerischen Werte zurück. Dieses Beispiel gibt den Gesamtumsatz aller in der Tabelle films aufgeführten Filme zurück.

SELECT SUM(revenue) AS total_revenue
FROM films;

9. AVERAGE ❗❗ #flashcard

AVERAGE berechnet den arithmetischen Mittelwert einer Spalte. Dieses Beispiel gibt den durchschnittlichen Umsatz aller in der Tabelle films aufgeführten Filme zurück.

SELECT AVERAGE(revenue) AS average_earned
FROM films;

10. WHERE ❗❗ #flashcard

Die WHERE-Klausel filtert Zeilen, die eine bestimmte Bedingung erfüllen. Zum Beispiel filtern wir unten Filme, die mehr als 500 Millionen Dollar eingenommen haben.

SELECT revenue 
FROM films
WHERE revenue > 500000000;

Andere bedingte Operatoren können ebenfalls verwendet werden, wie < (kleiner als), > (größer als), >= (größer oder gleich), <= (kleiner oder gleich), == (gleich), != (ungleich), um das Filtern durchzuführen.

11. HAVING ❗❗ #flashcard

Die HAVING-Klausel ähnelt der WHERE-Klausel, kann jedoch nur mit Aggregatfunktionen verwendet werden, während WHERE das nicht kann. In dem folgenden Beispiel wählen wir alle Filmgenres aus, die mindestens 50 Filme in ihrer Kategorie haben:

SELECT movie_name, director, date, COUNT(genre)
FROM films
GROUP BY genre
HAVING COUNT(genre) >= 50;

Hier ist ein weiteres Beispiel, das Filme nach Altersfreigabe gruppiert und nur die Freigaben auswählt, deren durchschnittlicher Umsatz über 100 Millionen liegt:

SELECT movie_name, director, date, AVG(revenue) as average_revenue
FROM films
GROUP BY age_rating
HAVING AVG(revenue) >= 100;

12. AND ❗❗ #flashcard

Der AND-Operator wird verwendet, um Zeilen zu filtern, die mehr als eine Bedingung erfüllen. Im folgenden Beispiel filtern wir nach englischsprachigen Filmen, die mehr als 500 Millionen Dollar eingenommen haben:

SELECT *
FROM films
WHERE revenue > 500000000 AND
     language == "English";

13. OR ❗❗ #flashcard

OR ist ein weiterer bedingter Operator, der es ermöglicht, Zeilen zu unterteilen, wenn eine der durch OR getrennten Bedingungen wahr ist. Dieses Beispiel gibt englischsprachige Filme zurück, die weniger als 100 Millionen Dollar eingespielt haben, oder französische Filme, die mehr als 500 Millionen Dollar eingespielt haben:

SELECT *
FROM films
WHERE (revenue < 100000000 AND language == "English") OR
     (revenue > 500000000 AND language == "French");

14. BETWEEN ❗❗ #flashcard

BETWEEN ermöglicht es, Zeilen innerhalb eines bestimmten Bereichs zu unterteilen, was WHERE-Klauseln einfacher und leichter lesbar macht. Im obigen Beispiel wählen wir alle Filme aus, die zwischen 2020 und 2022 veröffentlicht wurden:

SELECT *
FROM films
WHERE date BETWEEN 2020 AND 2022;

15. IN #flashcard

Der IN-Operator ist eine Abkürzung für mehrere OR-Anweisungen. Dieses Beispiel gibt alle Filme zurück, die in einem der Jahre 1998, 1966, 2001 und 2012 veröffentlicht wurden:

SELECT movie_name, date, revenue
FROM films
WHERE date IN (1998, 1966, 2001, 2012);

16. LIKE ❗❗ #flashcard

Der LIKE-Operator ermöglicht die Suche nach Mustern in einer Textspalte mithilfe von speziellen Zeichenketten, sogenannten Platzhaltern (Wildcards). Platzhalter ermöglichen das Finden von Textzeichenketten, die einem bestimmten Muster entsprechen. Zum Beispiel würde die Verwendung des Platzhalters "A% W%" in der Spalte director alle Regisseure finden, deren Vorname mit A beginnt und deren Nachname mit W beginnt.

SELECT *
FROM films
WHERE director LIKE "A% W%";

17. GROUP BY ❗❗ #flashcard

GROUP BY ermöglicht das Gruppieren von Zeilen basierend auf den Werten in einer Spalte. GROUP BY wird normalerweise mit Aggregatfunktionen wie COUNT, MIN, MAX, AVERAGE und mehr verwendet. Im folgenden Beispiel finden wir den durchschnittlichen Umsatz jedes Filmgenres.

SELECT AVG(revenue), movie_name, director, date
FROM films
GROUP BY genre;

Wir könnten auch MIN, MAX oder COUNT(revenue) verwenden, um den niedrigsten Umsatz, den höchsten Umsatz oder die Anzahl der Filme in jedem Genre zu finden. Beachten Sie, dass die GROUP BY-Anweisung die Abfrage praktisch nicht beeinflusst, wenn sie nicht mit einer Aggregatfunktion verwendet wird.

18. ORDER BY ❗❗ #flashcard

ORDER BY ermöglicht das Sortieren von Zeilen basierend auf einem Spaltenwert. Sie können nach aufsteigender (Standard) oder absteigender Reihenfolge sortieren, indem Sie ASC oder DESC hinzufügen. In diesem Beispiel wird der Umsatz in aufsteigender Reihenfolge sortiert.

SELECT *
FROM films
ORDER BY revenue ASC;

19. UPDATE ❗❗ #flashcard

UPDATE wird verwendet, um die Werte einzelner Zellen in einer vorhandenen Tabelle zu ändern. Es wird mit dem Schlüsselwort SET verwendet. In diesem Beispiel wird der Regisseur "J. Whedon" zu "Joss Whedon" aktualisiert.

UPDATE films
SET director = "Joss Whedon"
WHERE director = "J. Whedon";

Die WHERE-Klausel ist entscheidend, wenn UPDATE-Anweisungen geschrieben werden. Ohne sie würde die obige Abfrage alle Filme von Joss Whedon aktualisieren.

20. ALTER TABLE #flashcard

Die ALTER TABLE-Anweisung ermöglicht die Änderung der Eigenschaften der Tabelle und ihrer Spalten (nicht die tatsächlichen Zellwerte). Beispielsweise das Ändern von Spaltennamen, das Hinzufügen neuer Spalten, das Löschen von Spalten oder das Ändern ihres Datentyps. Die folgenden Beispiele zeigen das Löschen der Spalte date und das Hinzufügen der Spalte age_rating.

ALTER TABLE films
DROP COLUMN date;

ALTER TABLE films
ADD COLUMN age_rating;

21. CREATE TABLE ❗❗ #flashcard

CREATE TABLE erstellt eine neue Tabelle in einer Datenbank. Im folgenden Beispiel erstellen wir eine Tabelle "libraries" mit vier Spalten: eine Integer-Spalte namens id, eine Zeichenketten-Spalte namens name, eine Zeichenketten-Spalte namens version und eine Integer-Spalte namens num_downloads.

CREATE TABLE libraries (
   lib_id int,
   name varchar(100),
   version varchar(100),
   num_downloads int
);

22. INSERT INTO ❗❗ #flashcard

Die INSERT INTO-Anweisung kann verwendet werden, um neue Zeilen in eine Tabelle einzufügen. In diesem Beispiel fügen wir den Film "Doctor Strange" in die Tabelle films ein.

INSERT INTO films (movie_name, director, revenue, date, genre, language)
VALUES ("Doctor Strange 2", "Sam Raimi", 409000000, 2022, "Action", "English");

23. INNER JOIN #flashcard

Die INNER JOIN-Anweisung wählt Zeilen aus, die in beiden Tabellen übereinstimmende Werte haben. In der folgenden Abfrage verknüpfen wir die Tabellen films und soundtracks über eine gemeinsame Spalte "date", was Filme und Soundtracks zurückgibt, die im gleichen Jahr veröffentlicht wurden.

SELECT *
FROM films
INNER JOIN soundtracks
ON films.date = soundtracks.date
LIMIT 10;

24. LEFT JOIN #flashcard

Ein LEFT JOIN behält alle ursprünglichen Datensätze in der linken Tabelle bei und gibt fehlende Werte für alle Spalten aus der rechten Tabelle zurück, bei denen das Verknüpfungsfeld keine Übereinstimmung gefunden hat.

SELECT *
FROM films
LEFT JOIN soundtracks
ON films.date = soundtracks.date
LIMIT 10;

25. RIGHT JOIN #flashcard

Ein RIGHT JOIN behält alle ursprünglichen Datensätze in der rechten Tabelle bei und gibt fehlende Werte für alle Spalten aus der linken Tabelle zurück, bei denen das Verknüpfungsfeld keine Übereinstimmung gefunden hat. 

SELECT *
FROM films
RIGHT JOIN soundtracks
ON films.date = soundtracks.date
LIMIT 10;
