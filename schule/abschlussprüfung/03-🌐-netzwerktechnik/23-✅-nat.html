Netzwerkadressübersetzung - Network Address Translation (NAT)

TARGET DECK: [Schule] Prüfung::🌐Netzwerktechnik::23 ✅ NAT

Was ist Netzwerkadressübersetzung (Network Address Translation - NAT)? #flashcard

Die Netzwerkadressübersetzung (NAT) ist ein Verfahren, das es ermöglicht, eine einzige eindeutige IP-Adresse für eine gesamte Gruppe von Computern zu nutzen. NAT wird häufig auf Routern oder Firewalls implementiert und dient als Vermittler zwischen einem lokalen privaten Netzwerk und dem öffentlichen Internet.

Zweck von NAT #flashcard

Der Hauptzweck von NAT ist die Einsparung von öffentlichen IP-Adressen und die Verbesserung der Sicherheit. In der Vergangenheit waren IPv4-Adressen beinahe erschöpft, weshalb NAT eine Übergangslösung darstellt, um mit den begrenzten Adressen effizient umzugehen. Obwohl IPv6 dieses Problem langfristig lösen soll, bleibt NAT eine weit verbreitete Lösung.

Wie funktioniert NAT? #flashcard

NAT ermöglicht es einem einzigen Gerät, z. B. einem Router, als Vermittler zwischen dem öffentlichen und privaten Netzwerk zu fungieren. Geräte im privaten Netzwerk erhalten bei ausgehenden Verbindungen eine öffentliche IP-Adresse, die sie vor dem Rest des Internets verbirgt. Dies bietet zusätzliche Sicherheit und spart gleichzeitig Adressen ein.

NAT Beispiel #flashcard

Ein Host im privaten Netzwerk möchte eine Verbindung zum Internet herstellen. Der NAT-Router übersetzt die interne IP-Adresse des Hosts in eine globale IP-Adresse. Das eingehende Antwortpaket wird dann vom Router wieder in die ursprüngliche private Adresse zurückübersetzt und an den Host zurückgeleitet.

NAT-Konfiguration #flashcard

Eine typische NAT-Konfiguration erfordert:

  1. Ein Interface am Router für das externe Netzwerk (NAT Outside).
  2. Ein Interface für das interne Netzwerk (NAT Inside).
  3. Eine Regelmenge zur Übersetzung der IP-Adressen in den Paket-Headern.
    Das Netzwerk verwendet private, nicht registrierte IP-Adressen im internen Netzwerk, die durch NAT in öffentliche Adressen übersetzt werden, sobald eine Verbindung zum Internet hergestellt wird.

Multiplexing #flashcard

  • NAT-Überlastung nutzt die TCP/IP-Multiplexing-Funktion, um mehrere Verbindungen zu verschiedenen Hosts über dieselbe öffentliche IP-Adresse zu ermöglichen. Dies geschieht durch die Verwendung unterschiedlicher Ports.
  • Beispiel für eine IP-Header-Konfiguration:
    - Quelladresse: 192.168.0.1
    - Quellport: 620
    - Zieladresse: 52.220.51.237
    - Zielport: 2170

Vorteile von NAT #flashcard

  1. Adressersparnis: Reduziert den Bedarf an öffentlichen IP-Adressen.
  2. Sicherheit: Verbirgt die internen IP-Adressen und bietet eine zusätzliche Sicherheitsschicht.
  3. Flexibilität: Kann in verschiedenen Umgebungen eingesetzt werden, z. B. in drahtlosen Netzwerken.
  4. Skalierbarkeit: Funktioniert gut mit DHCP und erleichtert das Hinzufügen neuer Geräte.
  5. Multi-Homing: Ermöglicht mehrere Internetverbindungen und sorgt für Lastverteilung.

Nachteile von NAT #flashcard

  1. Ressourcenverbrauch: NAT benötigt Speicherplatz und Rechenleistung für die Übersetzung der Adressen.
  2. Verzögerungen: Die Adressübersetzung kann zu Verzögerungen führen.
  3. Einschränkungen: Einige Anwendungen und Protokolle funktionieren nicht korrekt mit NAT.
  4. Schwierigere Rückverfolgung: NAT kann die Nachverfolgung der Kommunikation erschweren, insbesondere bei der Verwendung von IPsec.

Arten von Network Address Translation (NAT) und deren Funktionsweisen #flashcard

Es gibt verschiedene Formen des Network Address Translation (NAT), die jeweils auf unterschiedliche Weise funktionieren. NAT dient im Wesentlichen dazu, private IP-Adressen in öffentliche IP-Adressen umzuwandeln, sodass Geräte in einem privaten Netzwerk mit dem öffentlichen Internet kommunizieren können. Dabei gibt es mehrere Methoden, wie NAT umgesetzt werden kann.

1. Statische Network Address Translation (SNAT) #flashcard

  • Die statische NAT (SNAT) ist eine Technologie, die eine bestimmte private (interne) IP-Adresse mit einer festen öffentlichen (externen) IP-Adresse verknüpft. Einfach gesagt: Es ist wie die Vergabe einer festen "Adresse" für ein Gerät im lokalen Netzwerk, damit es immer über die gleiche externe IP-Adresse auf das Internet zugreifen kann.
  • Diese 1-zu-1-Zuordnung ist besonders nützlich, wenn ein Gerät von außerhalb des privaten Netzwerks zugänglich sein muss, zum Beispiel bei Servern, die von externen Clients erreicht werden sollen.
  • Ein häufiges Anwendungsbeispiel sind Webserver, die eine feste öffentliche IP-Adresse benötigen, um kontinuierlich verfügbar zu sein.
  • Mit SNAT bleibt die IP-Adresse des internen Geräts statisch und unverändert, wenn es von außen erreichbar sein muss.

2. Dynamische Network Address Translation (DNAT) #flashcard

  • Im Gegensatz zur statischen NAT wählt die dynamische NAT (DNAT) eine IP-Adresse aus einem Pool von registrierten öffentlichen IP-Adressen, um eine private IP-Adresse zuzuordnen.
  • Bei DNAT kann jede nicht registrierte, private IP-Adresse in eine andere öffentliche IP-Adresse übersetzt werden, die dynamisch zugewiesen wird.
  • Diese Form der NAT eignet sich besonders für Netzwerke mit mehreren Geräten, die auf eine begrenzte Anzahl von öffentlichen IP-Adressen zugreifen müssen, da sie flexibler und effizienter in der Adressvergabe ist.

3. Reverse Network Address Translation (RNAT) #flashcard

  • Die Reverse Network Address Translation (RNAT) ermöglicht es Nutzern, sich über das Internet oder ein öffentliches Netzwerk mit sich selbst zu verbinden.
  • Dies wird in Szenarien genutzt, in denen ein Gerät innerhalb eines Netzwerks auf sich selbst zugreifen muss, um beispielsweise bestimmte Netzwerkdiagnosen durchzuführen oder Zugriffsanforderungen über das Internet umzuleiten.
  • RNAT kann auch in Fällen verwendet werden, in denen Benutzer den Datenverkehr innerhalb des gleichen Netzwerks über das Internet rückleiten müssen.

4. NAT Overload oder Port Address Translation (PAT) #flashcard

  • Eine sehr verbreitete Form der dynamischen NAT ist die NAT-Überladung, auch bekannt als Port-Level-Multiplexed NAT, Single Address NAT oder Port Address Translation (PAT).
  • Diese Technik verwendet verschiedene Ports, um mehreren privaten IP-Adressen (innerhalb des Netzwerks) den Zugriff auf eine einzige registrierte öffentliche IP-Adresse zu ermöglichen.
  • Mithilfe dieser Ports kann der NAT-Router den Netzwerkverkehr voneinander trennen und sicherstellen, dass die richtigen Daten an die richtigen Geräte weitergeleitet werden.
  • PAT wird oft verwendet, wenn viele Benutzer über eine einzige öffentliche IP-Adresse auf das Internet zugreifen, da es eine kostengünstige Methode ist, die den Bedarf an zusätzlichen IP-Adressen minimiert.
  • In PAT wird also für jede Verbindung zwischen einem internen Gerät und dem öffentlichen Internet ein eindeutiger Port zugewiesen. Das ermöglicht eine klare Unterscheidung des Netzwerkverkehrs, obwohl alle Verbindungen über eine gemeinsame öffentliche IP-Adresse laufen.

5. Overlapping NAT #flashcard

  • Overlapping NAT tritt auf, wenn zwei Organisationen, die beide IP-Adressen gemäß RFC 1918 (für private Netzwerke reservierte Adressbereiche) verwenden, ihre Netzwerke zusammenführen oder wenn eine registrierte IP-Adresse mehreren Geräten oder Netzwerken zugewiesen wird.
  • In solchen Fällen müssen die Netzwerke miteinander kommunizieren, ohne dass alle IP-Adressen der Geräte neu zugewiesen werden müssen.
  • Hier kommt NAT ins Spiel, indem es Adressen abfängt und sie entsprechend umsetzt.
  • Der NAT-Router führt eine Tabelle, in der er die IP-Adressen der Geräte speichert und sie in registrierte, eindeutige IP-Adressen übersetzt, um die Kommunikation zwischen den Netzwerken zu ermöglichen.
  • Diese Übersetzungen können entweder über statische NAT oder dynamische NAT erfolgen.

6. Der Stub-Domain-Kontext #flashcard

  • Im Kontext der Netzwerkadressübersetzung wird das interne Netzwerk oft als Stub-Domain bezeichnet.
  • Dies ist normalerweise ein lokales Netzwerk (LAN), das intern private IP-Adressen verwendet.
  • Der größte Teil des Datenverkehrs innerhalb einer Stub-Domain bleibt lokal, d.h. er verlässt nicht das interne Netzwerk.
  • Eine Stub-Domain kann sowohl registrierte als auch nicht registrierte IP-Adressen enthalten, wobei NAT dabei hilft, den Verkehr zwischen dem lokalen und dem öffentlichen Netzwerk zu vermitteln.

Carrier-grade NAT (CGNAT) #flashcard

  • Carrier-grade NAT (CGNAT), manchmal auch als Large Scale NAT (LSN) bezeichnet, ist eine Technik, die von Internetdienstanbietern (ISPs) verwendet wird, um die Knappheit von IPv4-Adressen zu bewältigen.
  • IPv4-Adressen sind begrenzt, und da die Anzahl der Geräte und Internetnutzer stetig wächst, reicht der verfügbare Pool an IPv4-Adressen nicht mehr aus. Um dieses Problem zu lösen, ermöglicht CGNAT es mehreren Kunden, gleichzeitig dieselbe öffentliche IP-Adresse zu nutzen, indem sie mit verschiedenen privaten IP-Adressen innerhalb ihres eigenen Netzwerks arbeiten.
  • CGNAT funktioniert ähnlich wie herkömmliches NAT, jedoch auf einer viel größeren Skala, da es auf der Ebene des Internetdienstanbieters implementiert wird und viele Endnutzer hinter einer einzigen öffentlichen IP-Adresse verbirgt.

Funktionsweise von CGNAT #flashcard

Bei CGNAT erfolgt die Adressübersetzung in zwei Stufen:

  • Im Heimnetzwerk des Nutzers: Der Router des Endnutzers übersetzt die privaten IP-Adressen der Geräte in eine private, jedoch vom ISP verwaltete IP-Adresse.
  • Beim Internetdienstanbieter: Der ISP nutzt CGNAT, um diese private IP-Adresse des Nutzers in eine gemeinsame öffentliche IP-Adresse zu übersetzen, die von vielen anderen Kunden gleichzeitig verwendet werden kann. Um den Datenverkehr korrekt zuzuordnen, verwendet der Anbieter Portnummern, um den eingehenden und ausgehenden Datenverkehr für jeden Nutzer zu identifizieren.

Herausforderungen von CGNAT #flashcard

  • Eingeschränkter Zugriff auf bestimmte Dienste: Da viele Nutzer eine gemeinsame öffentliche IP-Adresse verwenden, können bestimmte Dienste, die eindeutige IP-Adressen erfordern (wie z.B. Peer-to-Peer-Netzwerke oder einige Online-Spiele), Schwierigkeiten haben, korrekt zu funktionieren.
  • Sicherheits- und Datenschutzprobleme: Bei CGNAT können Aktivitäten mehrerer Nutzer unter einer einzigen IP-Adresse protokolliert werden, was die Rückverfolgung individueller Nutzeraktivitäten erschwert und potenziell rechtliche oder datenschutzbezogene Herausforderungen aufwirft.
  • Erschwerte Fehlerbehebung: Da mehrere Kunden dieselbe öffentliche IP-Adresse nutzen, kann es für Internetdienstanbieter schwierig sein, Netzwerkprobleme präzise zu diagnostizieren und spezifischen Nutzern zuzuordnen.

Vorteile von CGNAT #flashcard

Trotz der Herausforderungen bietet CGNAT eine effektive Übergangslösung für die aktuelle IPv4-Knappheit, während das Internet schrittweise auf das neuere IPv6-Protokoll umgestellt wird, das einen viel größeren Adressraum bietet. Für viele ISPs ist CGNAT eine notwendige Methode, um ihren Kunden weiterhin Internetdienste anbieten zu können, ohne sofort auf IPv6 wechseln zu müssen.
